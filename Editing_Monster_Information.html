5 Editing Monster Information

Monster information, including statistics and abilities is stored in a "monster database." See Step 4: Select a Monster Database for how the database is loaded into the game.

Interested in contributing to Frosthaven Manager? Help me build a monster database editor to make creating custom monster databases easier!

This section covers where and how to edit a monster database.

5.1 Where Can I Store Monster Information?

Simply: anywhere! Any plain-text file will do.

Note that some "text editors" are for rich text. Do not use Microsoft Word to edit plain-text files. On macOS, be careful with the TextEdit.app—it has options to edit rich-text format (RTF) files, which should be avoided when working with plain text.

Since the contents of the file will be in a format known as "Racket data," the traditional suffix is rktd. So you might create a plain-text file named my_monster_db.rktd to contain your monster database. We’ll cover the exact format in the next section, How can I Edit Monster Information?.

This is a bit long, but we can already identify several distinct components:

  1. Lines 2, 35, and 57 start definitions for "Monster Info."

  2. "Monster Info" contains many "Monster Stats," like on lines 5–12.

  3. Lines 23, 28, 78, and 83 start definitions for a "Monster Ability"

You might also notice a few other things: lots of parentheses, for one. These are good! They help us group things together. There is also a lot whitespace to help provide a visual feel of the layout. You are encouraged to use whitespace for layout, but the Frosthaven Manager will ignore it in most cases.

For the adventurous, game text obeys all the rules of Racket strings. Similarly, numbers can technically be written in any Racket number format.

You might notice double-quotes " around names like "guard" or "hynox archer". These denote game text, which is always enclosed by double-quotes to group the text together.

You might notice some numbers, like on line 5, are not in quotes. These are game numbers, like max HP. Many formats will do, but for Frosthaven Manager you should only see positive natural numbers, like 1, 2, 3, 42, etc. Contrast these with numbers written in quotes, like in "shield 3" on line 86: these numbers are part of the game text, like in an ability.

You might notice some symbols like #. The #s starts a definition, which is always followed by a group whose first element is the name of the group, like #s(monster-info). The #t and #f are better written as #true and #false, representing "true" and "false," respectively. These are game switches.

You might notice lines (like line 1) or parts of lines (like part of line 4) with semi-colons ; and english text. These are colored differently because they are comments: they help us humans understand the file, but they are ignored by the Frosthaven Manager when loading a monster database. This makes them handy to organize and structure databases.

With these observations, we’re ready to write a monster definition!

5.2.1.1 Write Your Own Monster

We’ll start with the monster information, which contains all the statistics needed for different levels of the monster. To begin, we write

#s(monster-info )

This indicates a monster information definition, as observed. Every part of the information goes inside the parentheses in a specific order.

The first two things after the monster-info label are the Set name and Monster name. For example, a Hobgoblin Warrior would have a Set name of "Warrior" and a Monster name of "Hobgoblin Warrior." Both of these are game text, so don’t forget the quotes:

#s(monster-info "Warrior" "Hobgoblin Warrior")

Next comes the normal and elite monster stats, so we need to add two groups. We’ll leave them empty for now. Let’s add comments and some space, too, so we know what’s what:

#s(monster-info "Warrior" "Hobgoblin Warrior"
                ;; Normal stats
                ()
                ;; Elite stats
                ())

Now we need to add the actual stats. These come in groups of eight, one for each possible monster level. Once you’ve seen one example of monster stats, you’ve seen most of them! We start with a definition:

#s(monster-stats )

The first three elements of monster-stats are max HP, base movement, and base attack. These are all game numbers. Let’s say the Hobgoblin Warrior has 4 max HP, move 2, and attack 2 at level 0:

#s(monster-stats 4 2 2)

The next three elements of monster-stats are persistent bonuses, attack effects, and condition immunities. All three are groups of game text, which means parentheses and quotes. Bonuses are things like "Shield 1." Attack effects are things like "2 Targets" and "Pierce 1." Condition immunities are things like "Poison." For many monsters, all three groups will be empty, written () like in the long example above. Let’s use the examples in this paragraph for the Hobgoblin Warrior, even though it might be far too strong for level 0, just to show how to write them:

#s(monster-stats 4 2 2
                 ;; persistent bonuses
                 ("Shield 1")
                 ;; attack effects
                 ("2 Targets"
                  "Pierce 1")
                 ;; condition immunities
                 ("Poison"))

Now you’ve seen it all: we need to write 16 total monster stats for the Hobgoblin Warrior, which we’ll do by making up the numbers as we go. Normally we would probably refer to an existing Frosthaven monster stats card, but you can also make your own.

#s(monster-info "Warrior" "Hobgoblin Warrior"
                ;; Normal stats
                (#s(monster-stats 4 2 2 () () ())
                 #s(monster-stats 5 2 2 () () ())
                 #s(monster-stats 6 3 3 () () ())
                 #s(monster-stats 7 3 3 () () ())
                 #s(monster-stats 8 3 4 () () ())
                 #s(monster-stats 9 3 4 () () ())
                 #s(monster-stats 10 4 5 () () ())
                 #s(monster-stats 11 4 5 () () ()))
                ;; Elite stats
                (#s(monster-stats 6 3 3 ("Shield 1") ("2 Targets" "Pierce 1") ("Poison"))
                 #s(monster-stats 7 3 3 ("Shield 1") ("2 Targets" "Pierce 1") ("Poison"))
                 #s(monster-stats 8 4 4 ("Shield 2") ("2 Targets" "Pierce 1") ("Poison"))
                 #s(monster-stats 9 4 4 ("Shield 2") ("2 Targets" "Pierce 1") ("Poison"))
                 #s(monster-stats 10 4 5 ("Shield 2") ("2 Targets" "Pierce 1") ("Poison"))
                 #s(monster-stats 11 4 5 ("Shield 2") ("2 Targets" "Pierce 1") ("Poison"))
                 #s(monster-stats 12 5 6 ("Shield 3") ("2 Targets" "Pierce 1") ("Poison"))
                 #s(monster-stats 13 5 6 ("Shield 3") ("2 Targets" "Pierce 1") ("Poison"))))

That’s an entire (if poorly-commented) monster definition for Hobgoblin Warriors. But right now it doesn’t have any ability cards!

Let’s write an ability card. As usual, we start with a definition:

#s(monster-ability )

Next we have the Set name to which the ability card belongs. Recall that, for example, archers all use the same deck. Here we want to use the "Warrior" set.

#s(monster-ability "Warrior")

Now we need a name for our card. Let’s go with "Crushing Blow."

#s(monster-ability "Warrior" "Crushing Blow")

All ability cards have an initiative, so let’s fill that in next. I think this is a slow ability, so let’s go with 65:

#s(monster-ability "Warrior" "Crushing Blow" 65)

All ability cards have a group of abilities, which are game text. I think Crushing Blow should have a short move and a big attack, so lets add two abilities for that:

#s(monster-ability "Warrior" "Crushing Blow" 65
                   ;; abilities on the card:
                   ;; - move
                   ;; - attack with muddle (to crush their spirits)
                   ("Move 2"
                    "Attack +4, Muddle"))

The last thing we need is to indicate whether this is a "shuffle" card or not—recall that some ability cards trigger a re-shuffle of the ability deck for that set. As usual, if the deck ever runs out, it is also re-shuffled. Crushing Blow doesn’t seem like the kind of ability that needs to shuffle the ability deck, so we’ll indicate the #false game switch, or no shuffle:

#s(monster-ability "Warrior" "Crushing Blow" 65
                   ;; abilities on the card:
                   ;; - move
                   ;; - attack with muddle (to crush their spirits)
                   ("Move 2"
                    "Attack +4, Muddle")
                   ;; no shuffle
                   #false)

If we wanted this card to trigger a shuffle, we would use #true instead.

You’ve now seen everything you need to write your own monster database. Below, you’ll find a succint reference for the format.

5.2.2 Monster Information Format Reference

The Developer Reference for monster information documents the underlying data structures, like monster-info, monster-stats, and monster-ability.

In the reference format below, names in <angle-brackets> refer to other rules in the reference, or to the following terms:

  • A definition is a group that starts with #s and whose first element names what is being defined.

  • A group is contained in (parentheses), with elements separated by whitespace.

  • Game text is contained in "double quotes".

  • A game number is a number with no extra decoration.

  • A game switch is either #true or #false.

  • Comments start with ; and run to the end of the line.

The use of ellipses ... indicate 0 or more of the preceding item.

;; Monster information
#s(monster-info
    ;; Set Name
    <game text>
    ;; Monster Name
    <game text>
    ;; Normal Stats for levels 0-7
    (<monster-stats>
     <monster-stats>
     <monster-stats>
     <monster-stats>
     <monster-stats>
     <monster-stats>
     <monster-stats>
     <monster-stats>)
    ;; Elite Stats for levels 0-7
    (<monster-stats>
     <monster-stats>
     <monster-stats>
     <monster-stats>
     <monster-stats>
     <monster-stats>
     <monster-stats>
     <monster-stats>))
 
;; Monster Stats
#s(monster-stats
    ;; Max HP
    <game number>
    ;; Base Movement
    <game number>
    ;; Base Attack
    <game number>
    ;; Persistent Bonuses
    (<game text> ...)
    ;; Attack Effects
    (<game text> ...)
    ;; Condition Immunities
    (<game text> ...))
 
;; Monster Ability
#s(monster-ability
    ;; Set Name
    <game text>
    ;; Card Name
    <game text>
    ;; Iniative
    <game number>
    ;; Abilities
    (<game text> ...)
    ;; Shuffle the ability deck?
    <game switch>)
